# HoMM3-Inspired Battle Game

I want to create a browser-based, turn-based tactical battle game inspired by the battle experience of Heroes of Might and Magic III.

The game should focus only on individual battles. A player should be able to prepare two armies, control one army, and fight against a computer-controlled opponent on a hex-based battlefield.

The armies should use human, non-magical units such as militia, pikemen, swordsmen, archers, crossbowmen, and cavalry. Different units should have different statistics and tactical strengths.

During battle, units should be able to move, attack in melee or at range where appropriate, wait, defend, and use simple physical abilities. Positioning, turn order, obstacles, retaliation, damage, and casualties should matter.

The game should clearly show the battlefield, armies, current turn, available actions, important unit information, and battle results. It should be understandable and usable without requiring knowledge of the original game.

Battles should run locally in the browser and should be reproducible for testing and debugging when the same setup and random seed are used.

The initial product should not include:

- A campaign or world map.
- Town or economy management.
- Heroes, magic, spells, artifacts, or supernatural units.
- Multiplayer.
- Accounts, a backend, or online persistence.
- Copied artwork, text, or other copyrighted assets from Heroes of Might and Magic.

The result should be an original game inspired by this style of tactical combat, not an exact recreation.

Use the v2 workflow to turn this request into a project blueprint and an initial playable-release plan. Do not implement the game yet.
